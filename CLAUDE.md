## Project Overview

Flask web application for OCR of vehicle license plates and odometer readings using Google Gemini Vision API. Captures images via camera or file upload, extracts text with AI, and manages vehicle records exportable to Excel.

## Commands

```bash
# Run locally (Windows)
run.bat

# Run manually
python app.py

# Install dependencies
pip install -r requirements.txt

# Production (Render)
gunicorn app:app

# Verify installation
python verificar_instalacion.py
```

No automated tests exist. QA is manual/browser-based.

## Architecture

**Two core backend files:**
- `app.py` — Flask routes, authentication, session-based vehicle storage, Excel export
- `ocr_processor.py` — `OCRProcessor` class wrapping Google Gemini 2.5 Flash for image-to-text

**Frontend (vanilla JS, no framework):**
- `templates/` — Jinja2 templates: `login.html`, `vehiculos.html` (dashboard), `captura.html` (camera capture)
- `static/js/captura.js` — Main capture logic: camera init, two-step workflow (plate → odometer), API calls
- `static/js/camera.js` — Legacy camera initialization
- `static/css/style.css` — All styling with CSS variables

**Key routes in app.py:**
- `POST /ocr/matricula` and `POST /ocr/cuentakilometros` — Accept Base64-encoded images, return JSON with extracted text
- `POST /agregar_vehiculo`, `/editar_vehiculo`, `/eliminar_vehiculo` — CRUD on session-stored vehicles
- `GET /descargar_excel` — Generate and stream .xlsx file

**Data flow:** Camera/upload → Base64 JPEG → Flask endpoint → decode to numpy array → `OCRProcessor` → Gemini API → cleaned text → JSON response

## Key Design Decisions

- **No database** — Vehicle data lives in Flask session cookies only (lost on logout)
- **Gemini-only OCR** — Tesseract and OCR.space were removed (see commit history); requires internet
- **HTTPS required for mobile** — Self-signed certs (`cert.pem`/`key.pem`) needed for camera access on phones
- **Authentication** — Simple credential pairs from `LOGIN_USERS` env var, protected by `@login_required` decorator
